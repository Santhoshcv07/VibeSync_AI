# Security Policy

## Reporting a Security Issue

Please do not publicly disclose suspected security vulnerabilities through public issues.

If a security issue is discovered, provide a clear description of:

- The affected area
- Steps to reproduce the issue
- The potential impact
- Any suggested mitigation

A private security contact method will be added before the project is publicly released.

## Security Principles

VibeSync AI follows these baseline rules:

- API keys and secrets must never be committed to Git.
- Real environment files must remain local.
- Example environment files must contain placeholders only.
- Provider API keys must remain on the backend.
- User input must be validated.
- Authentication and authorization must be enforced on protected resources.
- Logs must not expose passwords, tokens, API keys, or private user information.

## Current Status

VibeSync AI is under active development and has not yet reached a production security release.
