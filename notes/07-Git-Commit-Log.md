## Commit Rules

Use clear conventional commit messages.

Examples:

- `chore: initialize VibeSync AI repository`
- `feat: initialize Next.js frontend`
- `feat: initialize FastAPI backend`
- `docs: add project development notebook`
- `fix: correct backend startup configuration`
- `test: add recommendation rule coverage`

Never commit:

- API keys
- Passwords
- Tokens
- Real environment files
- Database credentials
- Broken builds
- Temporary debug output
